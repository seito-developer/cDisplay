# cDisplay - 仕様書

## 1. 概要

cDisplayは、macOSのデスクトップアプリケーションである。OSの解像度を変更せずに、画面上に黒いマスク（レターボックス/ピラーボックス）を重ねることで、任意のアスペクト比の疑似表示領域を作成する。

主な利用シーンは**映像制作プレビュー**と**配信・録画**である。OBSなどの画面キャプチャツールと組み合わせて使用する場合、マスクはフルスクリーンキャプチャの一部として取り込まれ、アスペクト比で整えられた映像出力が得られる。

## 2. 対象プラットフォーム

- **OS**: macOS 14（Sonoma）以降
- **配布**: Mac App Store（有料・買い切り）
- **言語**: 英語のみ
- **技術スタック**: Swift + AppKit

## 3. コア機能

### 3.1 マスク表示

- 選択されたアスペクト比に基づき、表示領域の外側を不透明な純黒のマスクで覆う。
- マスク領域は、メニューバー直下からの画面利用可能領域に基づいて計算する（メニューバーは覆わない）。
- 描画方式（複数のNSPanelを配置する方式、または単一の透明ウィンドウに不透明領域を描画する方式）は実装時に決定する。

### 3.2 アスペクト比プリセット

以下のプリセットを提供する：

| 名称           | 比率    | 主な用途                     |
|----------------|---------|------------------------------|
| ワイドスクリーン | 16:9    | 標準的な動画、配信           |
| スタンダード     | 4:3     | クラシックTV、レトロコンテンツ |
| シネマスコープ   | 2.39:1  | 映画的ワイドスクリーン       |
| スクエア         | 1:1     | Instagram、SNS向け           |
| バーティカル     | 9:16    | TikTok、Reels、Shorts        |

ユーザーはこれらのプリセットから選択する。カスタム比率の数値入力には対応しない。

### 3.3 オフセット位置

表示領域の画面内での配置をプリセットで調整できる：

- **中央**（デフォルト）
- **上寄せ**（表示領域が上端に配置され、マスクは下側のみ）
- **下寄せ**（表示領域が下端に配置され、マスクは上側のみ）

縦型アスペクト比（例：9:16）の場合、表示領域は常に水平方向中央に固定する。左右のオフセットプリセットは提供しない。

### 3.4 クリック動作（モード切替）

メニューバーのドロップダウンから切り替え可能な2つのモードを提供する：

- **クリック透過モード**（デフォルト）：マウスイベントがマスクを透過し、背後のウィンドウに到達する。マスクは純粋な視覚効果のみ。
- **クリック遮断モード**：マスク領域でのマウスイベントを遮断する。マスク上にカーソルを置くと、操作不可を示す禁止カーソル（NSCursor.operationNotAllowed）に変化する。

### 3.5 ガイドライン表示

マスクと表示領域の境界に、任意で白い細線（1pt）を表示できる。メニューバーからON/OFF切り替え可能。映像制作プレビュー時に有用であり、配信時は通常OFFにする想定。

### 3.6 フェードアニメーション

マスクのON/OFF切り替え時に**0.2〜0.3秒**のフェードイン/フェードアウトアニメーションを適用する。

マスク表示中のアスペクト比プリセット切り替えは、配信中に行うことを想定していない。プリセット切り替え時のトランジション動作は実装時に決定する。

## 4. ウィンドウ動作

### 4.1 表示順序（Z-order）

マスクウィンドウは**通常のアプリケーションウィンドウより上**、**システムUI要素（メニューバー、Spotlight、通知センター等）より下**に表示する。NSWindow.Level.floatingまたは同等のレベルを使用する。

### 4.2 対象ディスプレイ

- マスクは一度に**1つのディスプレイ**にのみ適用する（マスク有効化時に選択、デフォルトはメインディスプレイ）。
- 複数ディスプレイの同時マスクはスコープ外とする。

### 4.3 仮想デスクトップ（Spaces）

初期実装ではSpacesを特別に扱わない。マスクは現在のSpaceにおけるNSWindowのデフォルト動作に従う。

### 4.4 解像度・ディスプレイ変更時の動作

マスク表示中にディスプレイの解像度変更、または外部モニターの接続・切断が発生した場合、マスクは**自動的に無効化**される。ユーザーが手動で再度有効化する必要がある。

### 4.5 同一アスペクト比の検出

現在のディスプレイのアスペクト比が選択されたプリセットと一致する場合（マスク幅が0になる場合）、その旨をユーザーに通知し、マスクは適用しない。

## 5. ユーザーインターフェース

### 5.1 メニューバーアイコン

- アプリは**メニューバーステータスアイテム**（常駐アイコン）を表示する。
- アイコンは状態に応じて**外観が変化**する：
  - **マスクOFF時**：アウトライン/ワイヤーフレーム風アイコン
  - **マスクON時**：塗りつぶし/ソリッド風アイコン
- アイコンをクリックするとドロップダウンメニューが開く。

### 5.2 メニューバードロップダウンの内容

ドロップダウンメニューには以下の項目を含む：

1. **マスクON/OFF切り替え**（メイン操作）
2. **アスペクト比選択**（サブメニューにプリセット一覧）
3. **オフセット位置**（サブメニュー：中央 / 上寄せ / 下寄せ）
4. **クリックモード**（サブメニュー：クリック透過 / クリック遮断）
5. **ガイドライン**（トグル：表示 / 非表示）
6. **ディスプレイ情報**（マスクON時のみ表示）：
   - 現在のディスプレイ解像度
   - 表示領域のピクセルサイズ
7. **バージョン**（アプリのバージョン番号）
8. **終了**

### 5.3 Dockアイコン

- アプリは**Dockにアイコンを表示**する。
- Dockアイコンのクリックでマスクの**ON/OFFを切り替え**る。
- **設定ウィンドウは設けない**。全ての設定はメニューバーのドロップダウンから行う。

### 5.4 キーボードショートカット

- マスクON/OFFのトグル用に**デフォルトのグローバルキーボードショートカット**を提供する。
- キーの組み合わせは、一般的な配信ツール（OBS等）との競合を避けるものを選定する。
- ショートカットはメニューバーのドロップダウンからユーザーが変更可能とする。

## 6. 設定の永続化

- **アプリ終了後も保持する項目**：アスペクト比の選択、オフセット位置、クリックモード、ガイドライン表示状態、キーボードショートカット設定。
- **保持しない項目**：マスクのON/OFF状態。アプリは常にマスクOFFの状態で起動する。
- 設定はmacOS標準の仕組み（UserDefaultsまたはMAS互換のストレージ）で保存する。

## 7. オンボーディング

初回起動時に**簡単なオンボーディングツアー**（2〜3ステップ）を表示する：

1. マスクの切り替え方法（メニューバーアイコンとDockアイコン）
2. アスペクト比の選択と設定の調整方法
3. 画面キャプチャツールとの併用方法

ツアーは1回のみ表示する（永続化フラグで管理）。

## 8. アクセシビリティ

基本的なVoiceOver対応を行う：

- メニューバーのドロップダウン項目すべてに適切なアクセシビリティラベルを設定する。
- 状態変化（マスクON/OFF）をアナウンスする。

本格的なアクセシビリティ監査はMAS提出前のフェーズで実施する。

## 9. App Store・配布

- **ライセンスモデル**：有料・買い切り
- **Sandbox**：MAS要件として必須。本アプリは画面収録権限を必要としない（画面をキャプチャするのではなく、ウィンドウを重ねて表示するため）。
- **アプリ名**：cDisplay
- **アプリアイコン**：別途デザインする

## 10. 初期リリースのスコープ外

以下の機能は初期リリースから明示的に除外する：

- カスタムアスペクト比の数値入力
- 複数ディスプレイの同時マスク
- Spaces対応の動作
- ログイン時の自動起動
- マスク色のカスタマイズ（黒以外）
- マスク不透明度の調整
- ドラッグによるオフセット調整
- ローカライズ（英語以外）
- アプリ内アップデート通知
- 配信中のアスペクト比切り替えアニメーション
