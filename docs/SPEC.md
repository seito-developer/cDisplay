# cDisplay - 仕様書

## 1. 概要

cDisplayは、macOSのデスクトップアプリケーションである。ユーザーが選択したターゲット解像度に合わせて、**OSの表示解像度の変更**と**黒いマスク（レターボックス/ピラーボックス）のオーバーレイ**を組み合わせたハイブリッド方式で、任意のアスペクト比の表示領域を作成する。

主な利用シーンは**映像制作プレビュー**と**配信・録画**である。OBSなどの画面キャプチャツールと組み合わせて使用する場合、マスクはフルスクリーンキャプチャの一部として取り込まれ、アスペクト比で整えられた映像出力が得られる。

## 2. 対象プラットフォーム

- **OS**: macOS 14（Sonoma）以降
- **配布**: Mac App Store（有料・買い切り）
- **言語**: 英語のみ
- **技術スタック**: Swift + AppKit + CoreGraphics

## 3. コア機能

### 3.1 ハイブリッド表示方式

ターゲット解像度（例: 1920×1080）の適用時に、以下の3段階のフォールバックで最適な表示方法を自動選択する：

| 方式 | 条件 | 動作 |
|------|------|------|
| **解像度変更のみ** | ターゲットと完全一致するディスプレイモードが存在する | CoreGraphicsで解像度を切り替え。マスクなし |
| **解像度変更＋マスク補完** | ターゲットに近い解像度モードが存在する | 最近接モードに切り替え後、差分をマスクで補完 |
| **マスクのみ** | 適切なディスプレイモードが存在しない | 現在の解像度のまま、マスクオーバーレイで表示領域を制限 |

### 3.2 ターゲット解像度

ユーザーはアスペクト比ごとにグループ化されたターゲット解像度から選択する：

| アスペクト比 | ターゲット解像度 |
|-------------|-----------------|
| 16:9 | 1920×1080, 1600×900, 1280×720 |
| 4:3 | 1600×1200, 1280×960, 1024×768 |
| 2.39:1 | 1920×803 |
| 1:1 | 1080×1080, 720×720 |
| 9:16 | 1080×1920, 720×1280 |

### 3.3 マスク表示

- 選択されたターゲット解像度のアスペクト比に基づき、表示領域の外側を不透明な純黒のマスクで覆う。
- マスク領域は、メニューバー直下からの画面利用可能領域に基づいて計算する（メニューバーは覆わない）。
- 複数のNSPanelを配置する方式で描画する。

### 3.4 オフセット位置

表示領域の画面内での配置をプリセットで調整できる：

- **中央**（デフォルト）
- **上寄せ**（表示領域が上端に配置され、マスクは下側のみ）
- **下寄せ**（表示領域が下端に配置され、マスクは上側のみ）

縦型アスペクト比（例：9:16）の場合、表示領域は常に水平方向中央に固定する。

### 3.5 ガイドライン表示

マスクと表示領域の境界に、白い細線（1pt）を表示する。マスクパネルの表示領域側の端に描画される。

### 3.6 フェードアニメーション

マスクのON/OFF切り替え時に**0.25秒**のフェードイン/フェードアウトアニメーションを適用する。

### 3.7 クラッシュリカバリ

解像度変更時に元のモードIDをUserDefaultsに保存する。アプリがクラッシュした場合、次回起動時（UIの初期化前）に元の解像度を自動復元する。

## 4. ウィンドウ動作

### 4.1 表示順序（Z-order）

マスクウィンドウは**通常のアプリケーションウィンドウより上**、**システムUI要素（メニューバー、Spotlight、通知センター等）より下**に表示する。NSWindow.Level.floatingを使用する。

### 4.2 対象ディスプレイ

- マスクは一度に**1つのディスプレイ**（メインディスプレイ）にのみ適用する。
- 複数ディスプレイの同時マスクはスコープ外とする。

### 4.3 仮想デスクトップ（Spaces）

マスクは `collectionBehavior: [.canJoinAllSpaces, .stationary]` で全Spaceに表示される。

## 5. ユーザーインターフェース

### 5.1 メニューバーアイコン

- アプリは**メニューバーステータスアイテム**（常駐アイコン）を表示する。
- アイコンは状態に応じて**外観が変化**する：
  - **OFF時**: `rectangle.dashed`（SF Symbols）
  - **ON時**: `rectangle.fill`（SF Symbols）
- アイコンをクリックするとドロップダウンメニューが開く。

### 5.2 メニューバードロップダウンの内容

ドロップダウンメニューには以下の項目を含む：

1. **Enable / Disable**（メイン操作、状態に応じてラベルが切り替わる）
2. **Resolution**（サブメニュー: アスペクト比ごとにグループ化されたターゲット解像度一覧、選択中の項目にチェックマーク）
3. **ディスプレイ情報**（ネイティブ解像度、アクティブな表示方式の情報）
4. **バージョン**（アプリのバージョン番号）
5. **Quit cDisplay**

### 5.3 Dockアイコン

- アプリは**Dockにアイコンを表示**する（`activationPolicy: .regular`）。
- Dockアイコンのクリックで表示の**ON/OFFを切り替え**る。
- **設定ウィンドウは設けない**。全ての設定はメニューバーのドロップダウンから行う。

## 6. 設定の永続化

- **アプリ終了後も保持する項目**：選択されたアスペクト比、選択されたターゲット解像度（幅・高さ・ラベル）、選択されたディスプレイモードID、キーボードショートカット設定。
- **保持しない項目**：表示のON/OFF状態。アプリは常にOFFの状態で起動する。
- 設定はUserDefaultsで保存する。

## 7. アプリ終了時の動作

アプリ終了時に解像度変更が有効な場合、元の解像度に自動復元する。

## 8. App Store・配布

- **ライセンスモデル**：有料・買い切り
- **Sandbox**：MAS要件として必須。本アプリは画面収録権限を必要としない（画面をキャプチャするのではなく、ウィンドウを重ねて表示するため）。
- **アプリ名**：cDisplay
- **アプリアイコン**：別途デザインする

## 9. 初期リリースのスコープ外

以下の機能は初期リリースから明示的に除外する：

- カスタム解像度の数値入力
- 複数ディスプレイの同時マスク
- ログイン時の自動起動
- マスク色のカスタマイズ（黒以外）
- マスク不透明度の調整
- ドラッグによるオフセット調整
- ローカライズ（英語以外）
- アプリ内アップデート通知
- クリック遮断モード（マスク領域でのクリック遮断・禁止カーソル表示）
- オンボーディング（初回起動時のツアー）
- グローバルキーボードショートカット（イベントタップによるグローバルホットキー）
- ディスプレイ変更検知（解像度変更・外部モニター抜差し時の自動OFF）
- ガイドラインのUI切り替え（メニューからのON/OFFトグル）
- アクセシビリティ（VoiceOver対応）
